# 🚀 Deploying to Render

## 📋 Files Included for Render

- ✅ **Aptfile** - System dependencies (poppler, ffmpeg)
- ✅ **render.yaml** - Auto-configuration for Render
- ✅ **Procfile** - Backup deployment config
- ✅ **.env.example** - Environment variables template

---

## 🎯 Two Ways to Deploy

### Method 1: Automatic (Easiest) ⭐

**With render.yaml, Render auto-configures everything!**

1. **Push to GitHub:**
```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/yourusername/file-compressor.git
git push -u origin main
```

2. **Deploy on Render:**
- Go to [dashboard.render.com](https://dashboard.render.com)
- Click "New +" → "Web Service"
- Connect your GitHub repo
- **That's it!** Render reads `render.yaml` and configures everything automatically

3. **Done!**
- Render installs dependencies (from Aptfile)
- Builds your app
- Generates SECRET_KEY automatically
- Gives you: `https://your-app-name.onrender.com`

---

### Method 2: Manual Configuration

If render.yaml doesn't work or you want more control:

1. **Push to GitHub** (same as above)

2. **Create Web Service on Render:**
- Click "New +" → "Web Service"
- Connect GitHub repo

3. **Configure Settings:**

**Build Command:**
```
pip install -r requirements.txt
```

**Start Command:**
```
python app.py
```

**Environment Variables:**
```
SECRET_KEY=your-random-secret-key-here
PYTHONUNBUFFERED=1
```

4. **Deploy!**

## 💰 Free vs Paid Tier

### Free Tier
- ✅ 512MB RAM
- ⚠️ May struggle with large files or batch processing
- ⚠️ Sleeps after inactivity
- ✅ Good for testing

### Starter Tier ($7/month)
- ✅ 2GB RAM  
- ✅ Handles large files easily
- ✅ Batch processing works great
- ✅ Never sleeps
- ✅ Faster performance

**To switch to paid tier:**
- Edit `render.yaml`: change `plan: free` to `plan: starter`
- Or change in Render dashboard after deployment

## 🐛 Troubleshooting

### "Application failed to start"
**Check logs in Render dashboard:**
- Build logs: Did dependencies install?
- Deploy logs: Any Python errors?

**Common fixes:**
```bash
# Make sure requirements.txt has all dependencies
pip freeze > requirements.txt
git add requirements.txt
git commit -m "Update requirements"
git push
```

### "Out of memory"
- Upgrade to Starter tier ($7/month)
- Or limit file sizes in the app settings

### "Module not found"
- Check that all imports are in requirements.txt
- Redeploy to rebuild

### Video compression not working
- Check Aptfile has `ffmpeg`
- Check build logs show ffmpeg installation

## 📈 Performance Tips

### For Better Performance:
1. **Upgrade to Starter tier** - More RAM = faster processing
2. **Optimize file sizes** - Set max upload limits
3. **Use CDN** - For serving compressed files
4. **Enable compression** - Gzip responses

### In app.py, you can adjust:
```python
# Max file size
app.config['MAX_CONTENT_LENGTH'] = 100 * 1024 * 1024  # 100MB

# Processing threads
MAX_THREADS = 3  # Lower = less memory
```
